# Lab10 为 Lunch Tray 添加导航 实验报告
## 一、实验目的
基于 Jetpack Compose Navigation 组件，为 Lunch Tray 点餐应用实现多页面导航，掌握 NavController、NavHost、路由配置、返回堆栈管理及自适应顶部导航栏的开发方法。

## 二、核心组件关系说明
### 1. NavController、NavHost、composable() 三者关系
1. **NavController**：导航控制器，核心调度者，负责页面跳转、返回上一级、管理返回堆栈，提供 `navigate()`、`navigateUp()`、`popBackStack()` 等导航方法。
2. **NavHost**：导航宿主容器，作为页面承载容器，根据当前路由自动渲染对应页面，是所有可导航页面的载体。
3. **composable()**：导航路由注册函数，在 NavHost 内部为每一个页面绑定唯一路由字符串与对应 Compose 页面组件，构建整张导航图。

三者协作流程：NavHost 承载页面 → composable 注册所有路由 → NavController 控制路由切换与返回堆栈。

## 三、LunchTrayScreen 枚举类设计说明
本次实验使用 `enum class` 定义五个导航页面：Start、Entree、SideDish、Accompaniment、Checkout，每个枚举绑定字符串资源标题。

选用枚举而非硬编码字符串的原因：
1. **类型安全**：编译期校验路由名称，避免手写字符串拼写错误。
2. **统一管理**：所有页面路由集中在一个枚举类，便于维护和扩展。
3. **属性绑定**：可直接附带页面标题资源 ID，适配顶部 AppBar 动态标题。
4. **代码简洁**：可通过 `.name` 直接作为路由，通过枚举实例快速获取页面标题。

## 四、LunchTrayAppBar 设计思路
1. **动态标题**：接收当前枚举页面参数，自动读取枚举绑定的字符串资源，实现页面切换时标题同步更新。
2. **返回按钮控制**：通过 `navController.previousBackStackEntry != null` 判断是否有上一级页面，**Start 首页不显示返回按钮**，其余页面自动展示。
3. **返回功能**：点击返回箭头调用 `navController.navigateUp()`，遵循系统返回堆栈逻辑，回到上一页面。
4. **UI 样式**：适配 Material3 主题配色，保持应用视觉统一。

## 五、导航流程与返回堆栈设计
### 1. 页面导航流程
Start 开始点餐 → Entree 主菜 → SideDish 配菜 → Accompaniment 佐餐 → Checkout 结账。

任意页面点击 Cancel、结账页面点击 Submit，均直接返回首页并清空订单状态。

### 2. 返回堆栈管理策略
1. **进入点餐流程**：从 Start 跳转到主菜页面时，使用 `popUpTo` 将 Start 页面从返回堆栈弹出，用户后续按系统返回键直接退出应用，不会回到首页，符合常规 App 交互逻辑。
2. **取消/提交订单**：任意页面点击 Cancel 或 Submit 时，清空整个中间导航堆栈，直接回到 Start 首页，同时调用 ViewModel 重置订单数据，避免残留上一次选择的菜品。
3. **逐级返回**：顶部 AppBar 返回按钮遵循默认返回堆栈，逐级回退上一页面。

## 六、实验运行结果
1. 应用正常启动，默认展示 Start 首页，无返回按钮。
2. 点击 Start Order 可正常进入主菜选择页，顶部标题自动切换，显示返回箭头。
3. 依次点击 Next 可流畅完成：主菜→配菜→佐餐→结账页面跳转。
4. 任意页面点击 Cancel 可一键返回首页，订单数据清空。
5. 结账页面点击 Submit 返回首页，重置订单状态。
6. 从首页进入点餐流程后，按系统返回键直接退出应用，无多余页面回退。
7. 深浅色模式下导航栏与页面 UI 显示正常。

## 七、实验遇到的问题与解决方法
1. **问题**：页面跳转后返回堆栈混乱，按返回键会重复回到首页。
**解决**：使用 `popUpTo + inclusive` 配置导航路由，主动弹出不需要保留的页面，规范返回堆栈。

2. **问题**：点击 Cancel 返回首页后，上一次选择的菜品数据仍保留。
**解决**：导航回首页同时调用 `viewModel.resetOrder()`，清空订单 UI 状态。

3. **问题**：Start 首页顶部错误显示返回按钮。
**解决**：通过判断 `previousBackStackEntry` 是否为空，控制返回按钮显示隐藏。

## 八、实验总结
通过本次实验掌握了 Jetpack Compose Navigation 核心用法，理解了导航控制器、导航宿主、路由的协作逻辑，学会使用枚举管理页面路由、实现自适应顶部导航栏，同时掌握了返回堆栈的自定义管理方式，能够独立完成多页面 Compose 应用导航开发。
