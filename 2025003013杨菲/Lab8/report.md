# Lab8：构建超级英雄列表应用

## 一、实验简介
本次实验使用 Android Studio + Kotlin + Jetpack Compose 技术栈，综合运用数据类、静态数据源、资源文件管理、LazyColumn 可滚动列表、Material3 自定义主题、顶部应用栏、卡片布局等知识点，完成超级英雄展示 App。应用支持浅色/深色双主题适配，采用规范化组件布局，实现美观、可滚动的英雄列表界面。 

## 二、整体项目结构
项目采用分层模块化设计，结构清晰，职责分离：
1. **model 包**：存放数据类 Hero 与数据源 HeroesRepository，统一管理业务数据。
2. **ui/theme 包**：自定义 Material3 颜色、形状、字体、全局主题配置，适配深浅色模式。
3. **根目录页面文件**：
   - MainActivity.kt：程序入口，挂载全局主题与主界面。
   - HeroesScreen.kt：封装列表项、滚动列表、顶部导航栏核心 UI。
4. **res 资源目录**：
   - drawable：存放6个超级英雄图片资源。
   - font：引入 Cabin 自定义字体。
   - values：strings.xml 统一管理字符串文本资源。

## 三、核心模块设计说明
### 1. Hero 数据类设计
使用 Kotlin 数据类 `data class` 封装单个英雄信息，结合资源注解约束资源类型，避免传参错误。
- nameRes：英雄名称字符串资源ID。
- descriptionRes：英雄描述文本资源ID。
- imageRes：英雄头像图片资源ID。
采用资源ID绑定方式，方便多语言、主题统一管理，符合 Android 开发规范。

### 2. HeroesRepository 数据源
通过单例 `object` 构建静态仓库，集中初始化 6 个超级英雄对象，对外提供统一 List 集合。
优势：数据与 UI 解耦，后续新增、修改英雄信息只需修改数据源，无需改动页面代码。

### 3. 英雄列表项布局实现
单个英雄条目采用 `Card + Row` 水平布局完成：
1. 外层使用 Material3 Card 卡片，设置统一圆角样式，保证界面一致性。
2. 布局顺序：左侧文字区域（英雄名称+描述）、右侧英雄头像。
3. 文字使用 Column 垂直排列，区分标题与正文字号样式。
4. 头像固定尺寸、设置8dp圆角裁剪，使用 ContentScale.Crop 居中裁剪图片。
5. 加大卡片内边距与条目间距，优化视觉留白，使每个英雄展示框更大更美观。

### 4. 可滚动列表实现
通过 `LazyColumn` 实现高性能滚动列表：
- contentPadding 设置列表整体上下左右留白。
- verticalArrangement.spacedBy 控制每个卡片之间的间距。
- items 遍历数据源集合，批量复用 HeroItem 列表项组件。
LazyColumn 具备视图复用特性，适合长列表展示，减少内存占用。

### 5. Material3 自定义主题配置
1. **颜色**：分别定义浅色、深色两套配色方案，系统自动跟随手机主题切换。
2. **形状**：全局统一卡片、控件圆角规格，保证视觉统一。
3. **字体**：引入 Cabin-Regular 与 Cabin-Bold 自定义字体，全局替换默认字体，定制标题、正文文字层级与大小。
4. **全局主题**：通过 MaterialTheme 统一注入颜色、排版、形状，全局组件自动复用主题样式。

### 6. 顶部应用栏与状态栏适配
1. 使用 Scaffold 脚手架组件，快速整合 TopAppBar 顶部标题栏与主体内容。
2. 顶部应用栏居中展示应用名称，使用大标题字体样式。
3. 利用 innerPadding 自动避让顶部栏，防止列表内容被遮挡。
4. 适配系统状态栏，设置透明状态栏与图标明暗切换，深浅色模式下状态栏显示协调。

## 四、运行效果
1. 应用成功启动，顶部显示居中标题 Superheroes。
2. 完整展示 6 位超级英雄信息，列表支持上下流畅滚动。
3. 每个英雄卡片尺寸适中、布局规范，文字居左、头像居右。
4. 浅色模式界面干净清爽，深色模式自动切换暗色配色，文字清晰可读。
5. 图片圆角、文字层级、间距、圆角样式完全符合实验设计要求。

## 五、遇到的问题与解决方法
1. **字体文件报错**
问题：字体文件名包含中划线，Android 资源不合法。
解决：修改文件名为小写下划线命名格式，同步项目后正常识别。

2. **主题颜色变量缺失爆红**
问题：深色/浅色配色变量不完整，主题文件引用报错。
解决：补全全套深浅色颜色变量，统一 ColorScheme 绑定，消除代码红线。

3. **UI 布局样式不符**
问题：默认布局头像在左侧、卡片尺寸过小。
解决：调整 Row 排列顺序，加大内边距、间距与头像尺寸，完成头像居右、卡片放大的定制需求。

4. **内容被顶部栏遮挡**
解决：接收 Scaffold 内部边距参数，给列表添加内边距偏移，界面显示完整。
