# Lab14 公交时刻表应用实验报告

## 一、实验目的
1. 掌握 Jetpack Room 本地数据库框架的基本使用，完成数据表实体映射与数据访问开发
2. 学会加载 Assets 目录下预打包 SQLite 数据库，实现应用初始化读取本地预置数据
3. 理解 MVVM 架构思想，使用 ViewModel + Flow 实现界面与数据层解耦、状态监听
4. 熟练运用 Jetpack Compose 组件完成列表布局、顶部导航栏开发
5. 掌握 Navigation Compose 实现页面跳转、路由参数传递与页面返回功能
6. 完成公交时刻表应用整体开发，实现数据格式化、站点筛选、异常问题排查等功能

## 二、实验环境
- 开发工具：Android Studio
- 开发语言：Kotlin
- 界面框架：Jetpack Compose (Material3)
- 数据库框架：Jetpack Room
- 架构组件：ViewModel、Navigation Compose
- 异步与数据流：Kotlin Coroutine、Flow
- 运行设备：Android 模拟器

## 三、实验内容
1. 创建 BusSchedule 项目，按照 MVVM 分层思想搭建项目包结构
2. 定义 BusSchedule 数据实体类，完成与预置数据库表、字段的映射绑定
3. 编写 DAO 数据访问接口，实现查询全部班次、按站点筛选班次的 SQL 逻辑
4. 配置 Room 数据库类，采用单例模式并加载 assets 中的预打包数据库文件
5. 开发 ViewModel 层与 ViewModel 工厂类，负责数据中转与数据流对外暴露
6. 完成 Compose 主界面开发：
   - 顶部标题栏展示应用名称
   - LazyColumn 列表展示所有公交站点与发车时间
   - 对原始时间数据进行格式化，统一转为 HH:mm 标准格式
7. 实现列表点击交互，跳转至站点详情页面
8. 详情页根据路由参数筛选数据，仅展示当前站点的所有班次，支持页面返回
9. 配置 AndroidManifest.xml，完成自定义 Application 类注册，适配项目运行环境

## 四、核心实现
1. **数据库实体实现**
   - 使用 @Entity 注解绑定数据表名称，通过 @ColumnInfo 匹配数据库原生下划线字段
   - 声明主键，完成实体类与预打包数据表一一映射
2. **DAO 数据访问层实现**
   - 通过 @Query 注解编写 SQL 语句，实现全量数据查询、按站点名称条件查询
   - 接口方法返回 Flow 类型，实时监听数据库数据变化并通知 UI 更新
3. **Room 数据库配置**
   - 继承 RoomDatabase 创建数据库抽象类，声明获取 DAO 的抽象方法
   - 采用单例模式构建数据库实例，通过 createFromAsset 加载本地 db 文件
4. **ViewModel 层实现**
   - 接收 DAO 依赖对象，封装数据查询方法，对外暴露 Flow 数据流
   - 自定义 ViewModel 工厂类，实现带参 ViewModel 的依赖注入
5. **UI 与导航实现**
   - 使用 Scaffold + TopAppBar 搭建页面基础结构与顶部导航栏
   - LazyColumn 结合列表项布局展示公交时刻表，Divider 分割列表内容
   - 集成 Navigation Compose 配置双页面路由，完成参数传递与页面跳转
   - 编写时间格式化工具方法，将原始分钟数据转换为标准时分格式

## 五、运行结果
1. 应用启动后自动加载预打包数据库数据，首页正常展示全部公交时刻表信息
2. 列表布局整齐，发车时间成功格式化，以 HH:mm 格式正常显示
3. 点击任意站点列表项，可正常跳转至对应详情页面
4. 详情页自动筛选数据，仅展示当前站点的所有发车班次
5. 详情页左上角返回按钮功能正常，可一键回到主列表页面
6. 多次启停应用、切换页面运行稳定，无闪退、无数据错乱问题

## 六、问题与解决
1. 应用启动闪退：实体类字段名与数据库列名不匹配，添加 @ColumnInfo 注解手动绑定字段
2. 数据库加载失败：模拟器残留旧应用与缓存数据，卸载应用后重启模拟器重新运行
3. ViewModel 创建报错：带参 ViewModel 无法实例化，自定义 ViewModel 工厂类完成依赖注入
4. 路由参数异常：详情页获取路由参数未做容错，设置空值默认处理避免空指针
5. Compose 编译警告：使用实验性 Material3 组件，添加 @OptIn 注解消除警告
