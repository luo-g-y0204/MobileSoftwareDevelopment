# 掷骰子应用开发报告

## 一、应用界面结构说明
本应用基于 Android Jetpack Compose 实现，整体界面采用垂直居中结构：
### 1. 外层使用 Column 作为根布局，设置为全屏并居中显示；
### 2. 上方是 Image 组件，用于显示当前骰子点数的图片；
### 3. 中间使用 Spacer 留出固定间距，使布局更美观；
### 4. 下方是 Button 按钮，用于点击触发掷骰子逻辑。

整个界面简洁清晰，所有显示内容由状态数据驱动更新。

## 二、使用 Compose 状态保存骰子结果
骰子点数使用 Compose 的状态机制保存，核心代码：
var result by remember { mutableIntStateOf(1) }

### 1. mutableIntStateOf(1) 创建一个可观察的整型状态，初始值为 1；
### 2. remember 保证状态在界面重组时不会丢失，能够持续保存；
### 3. by 委托让我们可以像普通变量一样读写状态；
### 4. 当 result 发生变化时，Compose 会自动刷新相关界面。

## 三、根据点数切换图片资源
通过 when 表达式根据点数选择对应图片：
val imageResource = when(result) {
    1 -> R.drawable.dice_1
    2 -> R.drawable.dice_2
    3 -> R.drawable.dice_3
    4 -> R.drawable.dice_4
    5 -> R.drawable.dice_5
    else -> R.drawable.dice_6
}

每当 result 改变，when 会自动匹配新点数并返回对应图片 ID，Image 组件会自动加载新图片，实现图片切换。

## 四、断点设置与观察内容
### 1. 在按钮 onClick 点击事件内部设置断点
   观察按钮点击是否正常触发，随机数是否生成。

### 2. 在 result = (1..6).random() 处设置断点
   观察生成的随机数是否在 1~6 范围内。

### 3. 在 when(result) 入口处设置断点
   观察当前点数与图片资源是否匹配正确。

## 五、Step Into、Step Over、Step Out 使用体会
### 1. Step Into（跳入）
进入函数内部，适合查看系统方法或自己写的函数具体执行过程。

### 2. Step Over（跳过）
逐行执行代码，不进入函数内部，是最常用的调试操作。

### 3. Step Out（跳出）
直接执行完当前函数并跳出，回到调用处，适合快速跳过不需要细看的代码。

整体使用下来，三个调试功能配合可以清晰追踪程序运行流程，快速找到逻辑问题。

## 六、遇到的问题与解决过程
### 1. 问题：Unresolved reference 'R'
原因：包名不匹配，R 文件无法正确导入
解决：统一包名路径，正确导入项目 R 类。

### 2. 问题：DiceRollerTheme 报错找不到
原因：项目中没有该主题文件
解决：删除主题相关代码，直接加载界面。

### 3. 问题：按钮文字显示不对、大小不一致
原因：strings.xml 内容配置错误，按钮默认自适应大小
解决：修改字符串资源，给按钮设置统一最小尺寸。

### 4. 问题：点击按钮图片不刷新
原因：没用 Compose 状态管理，普通变量无法驱动 UI 更新
解决：使用 remember 和 mutableIntStateOf 管理骰子点数。

## 七、结论
### 1. 按钮点击后图片能自动刷新，是因为骰子点数是 Compose 可观察状态，状态改变后系统自动重组 UI。

### 2. 调试过程中，变量 result 的值与界面显示的图片完全一致，说明状态驱动逻辑正确，程序运行稳定。
