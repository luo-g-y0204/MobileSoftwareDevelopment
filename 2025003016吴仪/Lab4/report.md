## 二、布局结构说明
本次界面采用**两层嵌套布局**，结构清晰、层次分明：

1.  **最外层**：`Column` 作为根布局，通过 `Modifier.fillMaxSize().wrapContentSize(Alignment.Center)` 实现整体居中布局。
2.  **中间层**：垂直排列两个核心交互元素
    -   `Image`：根据随机结果动态加载对应骰子图片，实现点数展示
    -   `Button`：绑定点击事件，触发1-6随机数生成，更新界面状态
3.  **最内层**：`Image` 组件展示骰子面，`Button` 组件触发滚动逻辑

使用的核心组件：
-   布局组件：`Column` / `Spacer`
-   交互组件：`Button`
-   展示组件：`Image`（加载本地骰子资源）、`Text`（按钮文字）
-   状态管理：`remember` / `mutableStateOf`
-   样式修饰：`Modifier` 实现布局对齐、尺寸适配、间距调整

## 三、遇到的问题与解决过程

### 1. 图片资源引用错误、界面空白
-   **问题**：运行后骰子图片不显示，控制台报资源找不到错误，界面仅显示按钮
-   **解决**：
    1.  检查 `res/drawable` 目录，确认 `dice_1` 到 `dice_6` 图片资源完整且命名正确
    2.  修正 `when` 分支的资源引用，确保每个点数对应正确的 `R.drawable` 资源
    3.  补充 `contentDescription` 属性，保证无障碍适配

### 2. 点击按钮后界面无刷新
-   **问题**：点击「Roll」按钮，控制台打印随机数正常，但骰子图片无变化
-   **解决**：
    1.  确认状态变量 `result` 使用 `remember { mutableStateOf(1) }` 声明，确保状态变更触发重组
    2.  修正按钮点击事件，将 `result = (1..6).random()` 正确绑定到 `onClick` 回调
    3.  验证 `imageResource` 变量通过 `when` 表达式正确关联 `result` 状态，确保状态变更时图片同步更新

### 3. 布局居中效果异常
-   **问题**：骰子和按钮偏左显示，未在屏幕正中间
-   **解决**：
    1.  给 `Column` 布局添加 `horizontalAlignment = Alignment.CenterHorizontally`，实现子元素水平居中
    2.  给根布局添加 `Modifier.fillMaxSize().wrapContentSize(Alignment.Center)`，确保整个内容块在屏幕垂直、水平双居中
    3.  通过 `Spacer(modifier = Modifier.height(16.dp))` 调整图片与按钮的间距，优化视觉体验

### 4. 断点调试无法运行
-   **问题**：在代码行前面添加红色断点标记后，程序运行时无法在断点处暂停
-   **解决**：
    1.  使用**Debug模式**启动程序，点击工具栏小虫子图标，而非普通Run运行
    2.  断点位于按钮点击事件内部，必须点击界面上的Roll按钮才能触发断点
    3.  确认项目构建变体为Debug模式，保证调试功能正常开启
    4.  断点成功命中后，可在调试面板查看变量值，验证逻辑执行流程
