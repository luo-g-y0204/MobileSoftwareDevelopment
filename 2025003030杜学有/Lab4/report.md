## 一、应用界面结构说明
本应用基于 Kotlin 语言和 Jetpack Compose 框架开发，全程不使用 XML 布局文件，采用纯声明式 UI 构建。整体界面以 Column 垂直布局为核心容器，结构分为两个主要的可组合函数：

1. **DiceRollerApp**
作为应用的顶层组合函数，使用 Modifier.fillMaxSize() 填充整个屏幕，并通过 horizontalAlignment 和 verticalArrangement 将内部所有子组件设置为水平居中、垂直居中，为内部核心界面提供统一的布局环境。

2. **DiceWithButtonAndImage**
作为应用的核心交互界面，内部同样使用 Column 布局，包含三个关键组件：
- Image 组件：用于展示当前骰子点数对应的图片，根据点数动态切换；
- Spacer 组件：用于在图片与按钮之间设置固定间距，优化界面视觉效果；
- Button 组件：用于响应用户的点击操作，触发随机点数生成逻辑。

## 二、使用 Compose 状态保存骰子结果
应用通过 Jetpack Compose 提供的状态管理机制保存和维护骰子点数，核心实现代码如下：

```kotlin
var result by remember { mutableStateOf(1) }
```

各部分作用说明：
- mutableStateOf(1)：创建一个可观察的可变状态对象，设置骰子初始点数为 1；
- remember：用于包裹状态对象，保证在界面发生重组时，状态不会被重新创建，能够持续保存当前数值；
- by 关键字：使用 Kotlin 属性委托语法，简化状态对象的读写操作，可以直接通过变量名访问和修改数值。

当 result 变量被修改时，Compose 会自动检测到状态变化，并触发相关界面组件的重组，实现界面自动刷新。

## 三、根据点数切换图片资源
应用通过 when 表达式实现骰子点数到对应图片资源的映射，从而完成图片的动态切换，代码如下：

```kotlin
val imageResource = when (result) {
    1 -> R.drawable.dice_1
    2 -> R.drawable.dice_2
    3 -> R.drawable.dice_3
    4 -> R.drawable.dice_4
    5 -> R.drawable.dice_5
    else -> R.drawable.dice_6
}
```

实现逻辑：
1. 提前将 6 张不同点数的骰子图片资源放置在 res/drawable 目录下，文件名依次为 dice_1 到 dice_6；
2. 根据状态变量 result 的当前数值，通过 when 表达式匹配对应的图片资源 ID；
3. 将匹配得到的资源 ID 传入 Image 组件的 painterResource 中，实现图片展示。

每当点数 result 发生变化，imageResource 会同步更新，界面图片随之刷新。

## 五、Step Into、Step Over、Step Out 使用体会
1. **Step Over (F8)**
逐行执行当前代码行，不会进入函数或方法的内部，适合用于快速跟踪代码整体执行流程，直观查看每一步代码的执行效果。

2. **Step Into (F7)**
进入当前代码行所调用的函数或方法内部，能够深入查看函数的具体执行逻辑，例如可以进入 random() 方法查看随机数生成过程。

3. **Step Out (Shift + F8)**
跳出当前正在执行的函数或方法，返回到上一级调用位置，适合在深入函数内部后，快速回到主逻辑流程，梳理函数调用关系。

三种调试操作结合使用，可以完整、细致地观察程序从启动、界面构建到交互刷新的全过程。

## 六、遇到的问题与解决过程
1. **问题一：编译报错 Unresolved reference: DicerollerTheme**
原因：项目中缺少对应的主题文件，代码却引用了不存在的主题类；
解决方法：删除代码中所有关于 DicerollerTheme 的引用，直接加载界面组合函数，编译成功。

2. **问题二：设置断点后程序无法暂停**
原因：使用普通 Run 模式运行应用，而非调试模式；
解决方法：点击工具栏中的 Debug 'app' 按钮，以调试模式启动应用，断点正常触发。

3. **问题三：调试时状态变量显示为 result$delegate**
原因：Jetpack Compose 的状态对象通过 Kotlin 委托属性实现，属于正常显示形式；
解决方法：直接查看变量后面对应的数值，即为骰子的真实点数。
